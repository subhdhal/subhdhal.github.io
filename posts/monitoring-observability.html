Monitoring and Observability in Microservices

August 22, 2025

The Three Pillars of Observability

Observability in microservices is built on three fundamental pillars:

  • Metrics: Time-series data about system behavior
  • Logs: Event records of what happened in the system
  • Traces: Records of requests flowing through the system

Metrics with Prometheus

Prometheus scrapes metrics from your applications and services. Here's how to instrument your app: 

from prometheus_client import Counter, Histogram, start_http_server
import time

# Create metrics
request_count = Counter('app_requests_total', 'Total requests', ['method', 'endpoint'])
request_duration = Histogram('app_request_duration_seconds', 'Request duration')

@request_duration.time()
def process_request(method, endpoint):
    request_count.labels(method=method, endpoint=endpoint).inc()
    # Process request
    return "Success"

# Start Prometheus metrics server
start_http_server(8000)

Centralized Logging with ELK Stack

The ELK Stack (Elasticsearch, Logstash, Kibana) provides centralized logging: 

# Logstash configuration
input {
  tcp {
    port => 5000
    codec => json
  }
}

filter {
  if [type] == "app-logs" {
    grok {
      match => { "message" => "%{TIMESTAMP_ISO8601:timestamp} \[%{DATA:service}\] %{LOGLEVEL:level} %{GREEDYDATA:message}" }
    }
  }
}

output {
  elasticsearch {
    hosts => ["elasticsearch:9200"]
    index => "app-logs-%{+YYYY.MM.dd}"
  }
}

Distributed Tracing with Jaeger

Distributed tracing helps you understand the flow of requests through your microservices: 

from jaeger_client import Config

def init_jaeger_tracer(service_name):
    config = Config(
        config={
            'sampler': {
                'type': 'const',
                'param': 1,
            },
            'local_agent': {
                'reporting_host': 'localhost',
                'reporting_port': 6831,
            },
            'logging': True,
        },
        service_name=service_name,
    )
    return config.initialize_tracer()

tracer = init_jaeger_tracer('my-service')

with tracer.start_active_span('process_request') as scope:
    # Your code here
    pass

Alert Management

Define meaningful alerts based on SLOs: 

# Prometheus alert rules
groups:
  - name: service_alerts
    interval: 30s
    rules:
      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.05
        for: 5m
        annotations:
          summary: "High error rate detected"
          
      - alert: HighLatency
        expr: histogram_quantile(0.99, http_request_duration_seconds) > 1
        for: 10m
        annotations:
          summary: "High request latency detected"

Best Practices

  • Define SLOs and SLIs for your services
  • Use structured logging (JSON format)
  • Include correlation IDs in logs and traces
  • Monitor both application and infrastructure metrics
  • Alert on symptoms, not causes
  • Implement log retention policies
  • Use dashboards for visualization
  • Practice runbooks for alerts
  • Regularly review and update monitoring

Conclusion

Good observability is crucial for running reliable microservices. By implementing metrics, logging, and tracing, you gain the visibility needed to troubleshoot issues quickly and optimize your systems.
